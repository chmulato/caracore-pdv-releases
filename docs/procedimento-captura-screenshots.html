Procedimento de Captura de Screenshots

v3.1.1-free — Material técnico de apoio para treinamento

Aviso: Esta página documenta o processo técnico interno de captura de telas. A oferta pública atual está descrita em Release v3.1.1-free e Guia Comercial e Operacional.

Objetivo

Manter biblioteca de screenshots atualizados para material de treinamento, documentação e validação de interface do CaraCore-PDV. Este procedimento aplica-se à versão v3.1.1-free.

Pré-requisitos

  • Java JDK 21+ instalado
  • Maven 3.8+
  • Projeto caracore-pdv compilado e testado
  • Python 3.9+ (para captura automática)

Procedimento manual

  1. Compile o projeto: 
    mvn clean compile -DskipTests
  2. Execute a aplicação JavaFX: 
    mvn javafx:run
  3. Navegue pela interface e capture imagens em PNG com resolução 1920x1080 ou superior
  4. Salve as imagens em docs/assets/img/ com nomenclatura clara: caracore_pdv_tela_restrita_NN_descricao.png

Captura automática (opcional)

Para automação de screenshots com TestFX:

python scripts/capturador-screenshots.py \
  --backend auto \
  --upscale 2 \
  --output docs/assets/img

Telas principais para captura

  • Login: Tela de autenticação com erro e sucesso
  • Frente de Caixa: Operação de venda, busca de produtos, pagamento
  • Dashboard: Visão administrativa, vendas do dia, indicadores
  • Configuração Fiscal: Parâmetros de reforma tributária 2026
  • Segurança e Backup: Centro de controle local
  • Gestão de Produtos: Cadastro, NCM, alíquotas

Padrões de nomenclatura

Padrão Exemplo
caracore_pdv_tela_restrita_NN_descricao.png caracore_pdv_tela_restrita_02_operacao.png
caracore_pdv_tela_de_login.png caracore_pdv_tela_de_login.png
caracore_pdv_[funcionalidade].png caracore_pdv_tax.png, caracore_pdv_split_off.png

Verificação de qualidade

  • Resolução: Mínimo 1920x1080px para clareza
  • Formato: PNG (sem compressão com perda)
  • Conteúdo: Interface limpa, sem dados sensíveis ou PII
  • Atualização: Screenshots devem ser renovados com cada release significativa
