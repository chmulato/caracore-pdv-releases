Versão 3.0.1: correção de regressão no estoque

Atualização de manutenção da linha 3 (branch java_21). Corrige falha que impedia o registro correto de estoque no banco de dados após a migração inicial.

Versão estável atual: 3.0.1 — lançada em 21/04/2026. Se você usa a v3.0.0, recomendamos atualizar para esta versão para garantir a integridade do registro de estoque.

O que foi corrigido

  • Regressão no campo data_registro do estoque: a migração V027 (seed do vendedor padrão) usava DATE('now') do SQLite, que retorna apenas a data (YYYY-MM-DD). O Hibernate mapeava esse campo como LocalDateTime, esperando o formato YYYY-MM-DD HH:MM:SS, causando erro 400 nas vendas e 500 no dashboard.
  • Migração V028 adicionada: converte registros existentes com data incompleta para o formato correto (datetime(data_registro)), garantindo compatibilidade com instalações que já usavam a v3.0.0.
  • Migração V027 corrigida: substituído DATE('now') por datetime('now') para garantir o formato correto em novas instalações.

Impacto da atualização

Cenário Comportamento antes da v3.0.1 Comportamento após a v3.0.1
Nova venda com produto em estoque Erro 400 — requisição rejeitada Venda registrada normalmente
Dashboard Erro 500 — página não carregava Dados exibidos corretamente
Nova instalação Problema na criação do estoque padrão Estoque inicial criado com formato correto
Atualização de v3.0.0 — Flyway aplica V028 automaticamente na primeira inicialização

Como atualizar

  1. Baixe o instalador CaraCore PDV-3.0.1.exe na página de download.
  2. Execute o instalador — ele atualiza sobre a versão anterior sem perda de dados.
  3. Na primeira inicialização, o Flyway aplica a migração V028 automaticamente.
  4. Não é necessária nenhuma ação manual no banco de dados.

O banco de dados SQLite local é preservado integralmente durante a atualização.

O que permanece igual da v3.0.0

  • Todos os cadastros, vendas e histórico registrados na v3.0.0 são mantidos.
  • Motor fiscal IBS/CBS, PIX com split payment e dashboard gerencial sem alterações.
  • Perfis de operador, trilha de auditoria e configurações de loja inalterados.
  • Nenhuma reconfiguração necessária após a atualização.

Primeiro acesso (nova instalação)

  • Endereço local: http://localhost:8080
  • Usuário inicial: admin
  • Senha inicial: admin
  • No primeiro login há troca obrigatória de senha (8 a 15 caracteres)
  • Recomendado criar segundo operador administrativo para contingência
  • Banco: SQLite local (criado automaticamente via Flyway)
