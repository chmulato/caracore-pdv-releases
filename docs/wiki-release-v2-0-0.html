Release v2.0.0

Versão estável que consolidou continuidade operacional e previsibilidade para o dia a dia da loja.

Publicado em 09/04/2026. Marco de estabilidade com foco em venda sem interrupção, operação local segura e base pronta para crescimento.

Resumo para o cliente

  • Perfil: versão estável para operação de loja.
  • Valor entregue: menos risco de parada no caixa.
  • Canal: instalador oficial para Windows 10/11.
  • Modelo: operação local com dados sob controle da própria empresa.

Informações da aplicação

Domínio Definição na v2.0.0
Modelo de operação Aplicação local para varejo com acesso em localhost:8080
Persistência SQLite local como base oficial da entrega
Canal de distribuição Loja pdv.caracore.com.br — instaladores e wiki por versão
Objetivo da versão Consolidar estabilidade operacional da série 2.x

Regras de negócio centrais

  • RB-01 Continuidade operacional: o caixa deve continuar funcional em ambiente local.
  • RB-02 Canal estável: produção deve priorizar releases/latest.
  • RB-03 Integridade de release: instalador e metadados devem ser consistentes para update seguro.
  • RB-04 Fluxo essencial: login, cadastro, venda e recibo devem manter comportamento previsivel.

Conteúdo publicado

  • Instalador .exe
  • Arquivo latest.yml
  • Arquivo .blockmap para atualização incremental
  • Distribuicao centralizada no canal oficial de releases

Critérios de validação desta base

  • Instalação concluida sem erro em Windows 10/11
  • Inicialização do backend local em localhost:8080
  • Fluxo mínimo validado: login, cadastro de item, venda e recibo
  • Persistência local confirmada com SQLite

Situação no canal de releases

A release 2.0.0 permanece como base estável. A evolução incremental segue no canal de pré-release (2.0.1-rc1) para validações antes da próxima final.

Abrir instalador estável Abrir pré-release v2.0.1-rc1

Novidades desta versão (visão de negócio)

  • Mais confiança na rotina de atendimento em períodos de pico.
  • Relatórios mais estáveis para gestão e apoio ao contador.
  • Entrega padronizada para facilitar instalação e suporte.
  • Operação local reforçada para reduzir dependência externa.
  • Base mais madura para as evoluções das versões seguintes.

Funcionalidades por módulo

1) Acesso e operadores

  • Autenticação de usuário para operação de caixa e administração.
  • Base para organizacao de operadores por rotina da loja.

2) Cadastros mestres

  • Cadastro de produtos, clientes e entidades de apoio operacional.
  • Dados persistidos localmente no SQLite.

3) Ciclo de venda e recibo

  • Abertura e fechamento de venda com emissão de recibo digital.
  • Fluxo validado para uso real no contexto de degustação e loja.

4) Distribuicao e atualização

  • Publicacao centralizada com histórico de versões no canal oficial.
  • Suporte ao fluxo incremental de atualização.
