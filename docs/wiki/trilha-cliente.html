Handshake Inicial (Primeiro Acesso)

No primeiro uso, o CaraCore-PDV Desktop prepara o ambiente local. Isso não é “travamento”: é a fase em que ele coloca o caixa em modo operacional.

O que acontece (sem mistério)
  • Sobe o servidor local do PDV no seu computador.
  • Prepara o banco local (SQLite) para salvar vendas mesmo com internet instável.
  • Registra checks de primeira execução para diagnóstico (quando necessário).
  • Em instalação nova, cria o usuário padrão admin / admin (recomendado trocar depois).
Passo a passo (o que você faz)
  1. Baixe e execute o instalador.
  2. Abra o CaraCore-PDV Desktop.
  3. Aguarde a primeira inicialização (normalmente ~30 segundos).
  4. Faça login com admin / admin (primeira execução limpa).
  5. Se possível, altere a senha após o primeiro login.
O que é “offline” aqui

“Offline” significa que as vendas são salvas no seu computador usando SQLite. A internet pode cair e o caixa continua. Integrações automáticas (ex.: callback) dependem de internet.

Segurança e estabilidade: o banco local existe para garantir previsibilidade no balcão.
Quando você deve acionar suporte
  • Se ficar “iniciando” por tempo excessivo (ex.: mais de 2–3 minutos).
  • Se aparecer “Servidor local não iniciou”.
  • Se aparecer “Erro 02: …”.

Dica: quando aparecer uma tela de suporte, use o botão “Copiar diagnóstico para suporte” e cole no WhatsApp do suporte técnico.

Primeira Venda em Modo Offline

O objetivo aqui é validar o fluxo completo de balcão e confirmar que sua venda fica registrada localmente (SQLite).

Checklist rápido
  • Você conseguiu entrar no sistema.
  • Existe pelo menos 1 produto para vender (cadastro ou dados iniciais).
  • Você está no computador onde o Desktop está instalado.
Passo a passo (balcão)
  1. Acesse o menu de Vendas.
  2. Clique em Nova venda.
  3. Adicione itens (pesquisa ou leitor de código de barras, se configurado).
  4. Confira quantidades e total.
  5. Escolha a forma de pagamento (ex.: dinheiro/cartão/PIX manual).
  6. Clique em Finalizar.
  7. Confirme que a venda aparece registrada (lista/relatórios do dia).
Boa prática: faça primeiro uma venda pequena de teste. Depois, siga para operação completa.

Como interpretar mensagens do sistema

Mensagem não é “ameaça”. É o sistema te dizendo o que está acontecendo e qual o próximo passo seguro.

“Iniciando…” (primeiro acesso)

Normal no primeiro acesso: o Desktop prepara servidor local e banco SQLite.

O que fazer: aguarde ~30s. Se ultrapassar 2–3 minutos, acione suporte.

“Servidor local não iniciou”

O Desktop não conseguiu iniciar o servidor local automaticamente.

O que fazer: fale com o suporte e envie diagnóstico/logs (Trilha de Suporte).

“Erro 02: …” (integridade)

Proteção do sistema: ele bloqueia a abertura quando detecta estado local inconsistente.

O que fazer: use “Copiar diagnóstico para suporte” e envie ao suporte.

“SMTP não configurado” / falha de e-mail

E-mail depende de configuração. O PDV pode continuar funcionando, mas rotinas que enviam e-mail podem falhar.

O que fazer: se você precisa de e-mail, peça orientação ao suporte para configurar SMTP.

Dica anti-ansiedade

Quando algo dá errado, o que acelera a solução é evidência: print + diagnóstico + log. O sistema foi desenhado para gerar esse material rapidamente.
