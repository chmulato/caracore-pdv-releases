Manual Oficial de Upgrade (master -> java_21)

Guia operacional em HTML com capturas de tela para implantação segura do PDV.

Banco SQLite Flyway Rollback testado
1. Pré-check 2. Backup 3. Upgrade 4. Validação 5. Rollback

Passo 1 - Pré-check do ambiente

Execute os comandos abaixo e confirme Python e Maven centralizados.

& D:\\dev\\.venv\\Scripts\\python.exe --version
& D:\\dev\\.mvn\\activate-maven-env.ps1
mvn --version
Print 01: landing da aplicação PDV em localhost.

Passo 2 - Backup do banco

Crie backup do SQLite antes de qualquer alteração.

cd D:\\dev\\caracore-pdv
$timestamp = Get-Date -Format "yyyy_MM_dd_HHmmss"
Copy-Item .\\data\\caracore-pdv.db ".\\backups\\caracore-pdv_backup_$timestamp.db"
Print 02: seção "O que o CaraCore-PDV resolve".

Passo 3 - Executar upgrade + validação

Use o orquestrador Python para validar pré e pós-upgrade.

& D:\\dev\\.venv\\Scripts\\python.exe .\\scripts\\upgrade\\run_upgrade_and_validate.py ^
  --db-path .\\data\\caracore-pdv.db ^
  --backup-dir .\\backups ^
  --report-file .\\upgrade_validation_report.txt
Print 03: seção de benefícios do PDV.
Critério de aprovação: o relatório final deve conter result=GO.

Passo 4 - Validação operacional

  • Login no sistema
  • Consulta de venda
  • Teste de autorização por perfil
  • Validação de integridade do banco
Print 04: seção "Pronto para 2026" (fiscal).

Passo 5 - Rollback (se necessário)

Se houver qualquer falha crítica, execute rollback imediato:

& D:\\dev\\.venv\\Scripts\\python.exe .\\scripts\\upgrade\\rollback_from_backup.py ^
  --backup-dir .\\backups --latest ^
  --db-path .\\data\\caracore-pdv.db --force
Print 05: seção final da página principal da aplicação.
NO-GO: qualquer inconsistência de dados, erro de startup ou falha de segurança.
