Versão 3.0.10: entrega portátil Windows e CI/CD híbrido

Atualização da linha 3 (branch java_21). Consolida o pipeline de entrega contínua com fallback automático JVM, Electron dual-mode e publicação estável de releases no repositório público.

Versão estável atual: 3.0.10 — lançada em 23/04/2026. Se você usa v3.0.1 ou anterior, recomendamos atualizar para esta versão para obter o instalador ZIP portátil mais recente.

O que foi adicionado

  • Entrega portátil Windows (CI/CD): workflow híbrido no GitHub Actions — tenta build Quarkus Native; em caso de falha, aciona fallback automático para empacotamento JVM (fast-jar). Garante instalador ZIP estável em todas as máquinas Windows.
  • Electron dual-mode: electron/main.js detecta automaticamente *-runner.exe (native) ou quarkus-run.jar (JVM) e inicializa o backend correto, com JRE embutido para a rota JVM. O usuário não precisa instalar Java separadamente.

O que foi corrigido

  • Erro 403 na publicação de releases: token ausente ou incorreto em PUBLIC_RELEASES_TOKEN impedia a publicação cruzada no repositório público caracore-pdv-releases — corrigido com permissão Contents: Read and write e PAT explícito no workflow.
  • Publicação via PAT explícito: release.yml agora usa PUBLIC_RELEASES_TOKEN diretamente em softprops/action-gh-release, eliminando dependência do token padrão do repositório.

Impacto para o usuário final

Cenário Antes da v3.0.10 Após a v3.0.10
Download do instalador Artefato manual; release poderia falhar silenciosamente ZIP publicado automaticamente a cada tag via CI/CD
Instalação em Windows sem GraalVM Build nativo falhava; nenhum pacote gerado Fallback JVM acionado automaticamente; ZIP sempre disponível
Execução via Electron Somente modo native; erros se runner.exe ausente Modo dual: detecta native ou JVM automaticamente
Java pré-instalado Necessário na rota JVM JRE embutido no ZIP — Java não é necessário no sistema

Como atualizar

  1. Baixe o instalador CaraCore PDV-3.0.10.exe (ou ZIP portátil) na página de download.
  2. Execute o instalador — ele atualiza sobre a versão anterior sem perda de dados.
  3. Na primeira inicialização, o Flyway aplica migrações pendentes automaticamente.
  4. Não é necessária nenhuma ação manual no banco de dados.

O banco de dados SQLite local é preservado integralmente durante a atualização.

O que permanece igual das versões anteriores

  • Todos os cadastros, vendas e histórico registrados nas versões anteriores são mantidos.
  • Motor fiscal IBS/CBS, PIX com split payment e dashboard gerencial sem alterações funcionais.
  • Perfis de operador, trilha de auditoria e configurações de loja inalterados.
  • Nenhuma reconfiguração necessária após a atualização.

Primeiro acesso (nova instalação)

  • Endereço local: http://localhost:8080
  • Usuário inicial: admin
  • Senha inicial: admin
  • No primeiro login há troca obrigatória de senha (8 a 15 caracteres)
  • Recomendado criar segundo operador administrativo para contingência
  • Banco: SQLite local (criado automaticamente via Flyway)
